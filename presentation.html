03 — Deep Dive
15단계 분석 · 12개 에이전트 · 6개 워크플로우, 무엇을 하는가
harness-ito는 코드를 한 번 훑는 도구가 아니라, 프로젝트 지식을 구조화하고 역할별 AI 협업 체계를 만드는 하네스입니다.

🔍 15단계 자동 분석

코드, SQL, 트랜잭션, 외부 연동, 컨벤션을 인덱스로 만들어 AI가 근거를 보고 답하게 합니다.
1. 프로젝트 스캔언어·프레임워크·모듈 구조 파악
2. 엔트리포인트Controller, Action, Batch, API 추출
3. 심볼 인덱스클래스·메서드·DTO·Mapper 목록화
4. 호출 그래프직접/간접 호출 경로 생성
5. SQL 사용처MyBatis/iBatis/JDBC/PLSQL 매핑
6. DB 스키마 연결테이블·컬럼·CRUD 관계 정리
7. 트랜잭션 경계@Transactional, SP, commit/rollback 추적
8. 외부 I/OHTTP, 파일, 메시지, 배치 연동 탐지
9. 화면/라우팅JSP, Vue, URL, 메뉴 연결
10. 컨벤션 추출Controller-Service-DAO 작성 패턴 학습
11. 위험 패턴N+1, 풀스캔, Null, 권한 누락 후보
12. 레거시 후보장기 미사용·중복·데드코드 후보
13. 테스트 후보회귀 테스트와 목 데이터 추천
14. 문서 동기화README, 업무 흐름, 용어 차이 탐지
15. 하네스 생성인덱스·패턴·에이전트 프롬프트 구성

05 — Demo ②
레거시 역공학 — 3천 줄 PL/SQL, 5분 만에 해석
원개발자 없이도 OK. AI가 의도·흐름·위험 포인트까지 자동 설명
AS-IS — 원본 PL/SQL 해석 불가
SP_PROCESS_ORDER.sql (3,241 lines)

PROCEDURE SP_PROCESS_ORDER(
P_ORDER_ID IN NUMBER,
P_STATUS IN VARCHAR2,
P_RESULT OUT NUMBER) IS
V_CNT NUMBER := 0;
V_LOCK_YN CHAR(1);
-- TODO: 나중에 정리 (2016)
BEGIN
SELECT NVL(LOCK_YN,'N') INTO V_LOCK_YN
FROM TB_ORDER_CTL WHERE ROWNUM=1;
IF V_LOCK_YN = 'Y' THEN
P_RESULT := -999; RETURN;
END IF;
-- 이하 2,800줄...
AI 자동 해석 결과 즉시 이해 가능
📋 프로시저 목적
수주 주문 처리 메인 프로시저. 수주 유효성 검증 → 재고 차감 → 생산 지시 생성 → 정산 연동까지 단일 트랜잭션으로 처리.
🔀 핵심 흐름 (7단계)
1. 잠금 체크 TB_ORDER_CTL.LOCK_YN = 'Y'면 즉시 -999 반환
2. 중복 수주 방지 동일 ORDER_ID 존재 시 차단
3. 재고 차감 TB_STOCK UPDATE → 부족 시 ROLLBACK
4. 생산 지시 INSERT TB_PROD_ORDER 생성
5. 정산 큐 등록 TB_SETTLE_QUEUE INSERT
6. 감사 로그 TB_AUDIT_LOG 기록
7. P_RESULT = 0 성공 반환
⚠️
발견된 위험 포인트: 3번 재고 차감 후 5번 정산 큐 실패 시 재고가 복원되지 않는 버그 존재. SAVEPOINT 없이 부분 ROLLBACK 불가 구조.

11 — PoC Plan
바로 검증 가능한 PoC 범위와 다음 액션
큰 전사 도입이 아니라, 대표 프로젝트 1개로 2주 안에 효과와 리스크를 함께 검증합니다.

🎯 PoC 범위

대상
레거시 성격이 강하고 변경 요청이 잦은 대표 업무 프로젝트 1개
예: Struts/Spring + MyBatis/iBatis + Oracle/PLSQL 조합
기간
2주: 준비 2일 → 하네스 생성 3일 → 실제 업무 케이스 검증 5일 → 결과 정리 2일
검증 업무
영향도 분석 2건, SQL 리뷰 2건, 레거시 해석 1건, 신규 기능 스캐폴딩 1건
산출물
call_graph, symbols, sql_usage, schema, transactions, external_io, 컨벤션 문서, 에이전트별 결과 리포트
제외 범위
운영 코드 자동 배포, 무검토 자동 수정, 전사 인덱스 공유는 PoC 범위에서 제외
